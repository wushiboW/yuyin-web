# 屿音电商前端设计开发规范

> 本文件从主仓 `Yuyin-E-commerce-platform-fe` 的历史中恢复,作为 `wushiboW/yuyin-web` (Web 端) 的开发守则。修改请同步至主仓 `front-end/` 共享或 `yuyin-app/` 协调门户。

---

基于梵几、之禾、木墨、ZENS、Prada、Machine-A 等高端品牌官网的深度分析，结合屿音电商最新的「C 端统一前台 + B 端独立中台」前端架构，定制的专属设计规范，用于指导所有前端应用的页面设计与开发。

## 一、核心设计原则

### 1.1 品牌优先，调性一致
所有分析的中高端品牌都将品牌调性放在首位，网站设计完全服务于品牌表达，而非单纯的交易工具：
- **色彩系统**：严格遵循品牌 VI 体系，主色调控制在 3-5 种以内，避免繁杂色彩干扰品牌认知。例如梵几采用暖灰 + 大地色系还原自然质感，Prada 采用黑白灰极简配色传递轻奢感，ZENS 哲品则以原木色传递东方生活美学。
- **字体系统**：优先使用品牌定制字体，或根据品牌气质选择匹配的字体：奢侈品品牌常用 Didot、Bodoni 等衬线字体传递高级感，家居品牌常用简洁无衬线字体传递自然亲和感。
- **视觉延伸**：将品牌符号延伸到页面设计中，如梵几从竹椅椅背提取平行四边形作为辅助图形，通过重复排列形成视觉韵律，强化品牌记忆点。

### 1.2 极简克制，留白为王
高端品牌普遍采用极简设计哲学，通过大量留白突出核心内容，避免信息过载：
- **去营销化**：首页不强制展示所有商品和促销信息，而是分模块呈现品牌故事、产品系列、生活方式内容。如梵几、之禾首页都以品牌叙事为主，而非堆满商品列表，没有喧嚣的折扣弹窗。
- **呼吸感布局**：采用 8pt 网格系统，模块之间保留充足间距，让页面有自然的呼吸感，避免拥挤。例如 Prada 官网商品居中全屏展示，周围大量留白，让产品成为绝对焦点。
- **少即是多**：去除不必要的装饰元素，只保留核心功能和内容，让用户的注意力完全集中在产品和品牌本身。

### 1.3 视觉层级，信息有序
通过清晰的视觉层级引导用户浏览路径，让用户自然获取信息：
- **优先级区分**：通过字体大小、字重、颜色明确信息优先级：标题 > 副标题 > 正文 > 辅助信息。首页遵循统一的信息排序：顶部 LOGO + 高频操作（搜索、购物车），中部品牌 Banner > 分类导航 > 精选商品 > 品牌内容。
- **模块化拆分**：将内容拆分为独立的功能模块，每个模块聚焦一个主题，如梵几的家具、家居、艺术栏目，清晰区分不同业务板块，用户可以快速定位自己感兴趣的内容。

### 1.4 移动优先，响应适配
所有品牌都采用移动优先的设计思路，确保全设备体验一致：
- **小屏优先**：先设计移动端布局，再向大屏扩展，确保移动端体验不打折扣，而非简单的桌面端缩小。
- **自适应调整**：模块在不同屏幕下自动折叠、缩放，移动端简化复杂导航，突出搜索、购物车等核心功能，保证单手操作的便利性。

### 1.5 全流程无缝体验
基于 C 端统一前台架构，用户从浏览商品、加购、下单、支付到查看订单、管理账户的全流程，都在同一个应用内无缝完成，无跨应用跳转、加载延迟，打造沉浸式的用户体验。

## 二、页面布局规范

### 2.1 导航栏设计
- **固定顶部通栏**：采用固定顶部导航，确保用户随时可以访问核心功能，布局遵循：LOGO 居左，导航菜单居中 / 居右，搜索、购物车、用户入口居右的经典结构。
- **精简导航层级**：高端品牌导航选项精简，避免过多下拉层级，确保用户最多 3 次点击即可到达目标页面，降低用户的操作成本。
- **移动端适配**：移动端将复杂导航折叠为汉堡菜单，保留搜索、购物车等高频入口，避免小屏下导航拥挤。

### 2.2 首页布局
- **首屏沉浸式 Banner**：首屏采用大幅高清品牌形象图或视频，传递品牌调性，如之禾的季节系列形象片、Prada 的动态视觉，让用户第一眼就感受到品牌的气质。
- **模块式流式布局**：从上到下遵循统一的内容流：品牌 Banner > 核心系列 / 分类导航 > 精选商品展示 > 品牌故事 / 内容栏目 > 线下门店 / 购买渠道，符合用户的浏览习惯。
- **差异化内容侧重**：根据品牌定位调整内容侧重：家居品牌侧重生活方式内容，时尚品牌侧重新品系列，买手店侧重品牌集合展示。

### 2.3 商品列表页
- **响应式网格布局**：采用统一的响应式网格，桌面端 3-4 列，平板 2-3 列，移动端 1-2 列，所有商品卡片保持统一尺寸，保证视觉整齐。
- **极简商品卡片**：商品卡片只展示核心信息：高清主图、商品名称、价格，避免多余信息干扰。鼠标 hover 时显示快速操作（加入购物车、查看详情），提升操作效率。
- **精简筛选功能**：侧边或顶部筛选栏，支持按品类、价格、颜色等核心维度筛选，高端品牌避免过多筛选条件，保持界面简洁。

### 2.4 商品详情页
- **大图优先展示**：左侧大幅商品图区域，支持多图切换、细节放大，高端品牌会提供多场景图、材质细节图，充分展示产品的工艺和质感。
- **信息分层展示**：右侧信息从上到下遵循统一的优先级：商品名称 > 价格 > 规格选择 > 加入购物车按钮 > 商品详情 > 品牌故事，让用户逐步获取信息。
- **克制的营销干扰**：避免过多的促销弹窗、关联推荐，突出商品本身的信息，如 Prada 详情页只展示核心商品信息，没有多余的干扰元素。

## 三、交互设计规范

### 3.1 微交互与动效
- **轻量动效原则**：优先使用 CSS 过渡动画，避免过重的 JS 动画，动效的目的是提升体验而非炫技。例如 hover 时的颜色变化、图片淡入淡出，subtle 而不突兀。
- **滚动渐显效果**：内容随用户滚动逐步加载显示，增加页面的活力，但不干扰用户的浏览节奏，如梵几官网的滚动渐显效果，让页面更有层次感。
- **动态视觉过渡**：高端品牌会采用静态到动态的过渡效果，如 Prada 官网鼠标悬停在商品图上时自动播放短视频，让用户更直观的感受产品，提升沉浸感。

### 3.2 加载状态设计
- **骨架屏占位**：内容加载时使用骨架屏占位，减少用户的等待焦虑，避免白屏时间过长。
- **懒加载机制**：非首屏的图片、内容采用懒加载，优先加载首屏核心内容，提升首屏加载速度，减少带宽消耗。
- **操作反馈**：按钮点击后显示加载状态，告知用户操作正在进行，避免用户重复点击。

### 3.3 表单交互设计
- **精简表单字段**：注册、结算表单只收集必要信息，减少用户的输入成本，高端品牌避免强制用户注册，支持游客购买。
- **实时输入验证**：用户输入时实时验证格式，错误提示清晰友好，避免用户提交表单后才发现错误。
- **自动填充支持**：支持地址、支付信息的自动填充，提升结算流程的效率，减少用户的操作步骤。

## 四、前端开发规范

### 4.1 组件化开发
- **原子化组件拆分**：将页面拆分为独立、可复用的原子组件，遵循单一职责原则，每个组件只负责一个功能，如按钮、输入框、商品卡片、导航栏等。
- **低耦合高内聚**：组件之间通过 props 传递数据，通过事件触发交互，严禁在组件内部直接操作全局 DOM 或修改父组件状态，保证组件的独立性。
- **统一设计系统**：建立完整的设计系统，统一所有组件的样式、交互、尺寸，确保整个网站的视觉和交互一致性，同时提升开发效率。

### 4.2 代码规范
- **语义化 HTML**：使用语义化标签（header、nav、main、section、footer、article），提升页面的可访问性和 SEO 友好性，避免通篇 div。
- **样式管理**：采用 CSS Modules 或 Tailwind CSS 等方案，避免样式冲突，统一使用 BEM 命名规范，保证样式的可维护性。
- **注释与文档**：关键业务逻辑、组件 API 添加清晰的注释，公共组件编写使用文档，便于团队协作和后续维护。

### 4.3 性能优化规范
性能是电商网站的生命线，直接影响用户留存和转化率，需要遵循以下优化措施：

**图片优化**
- 使用 WebP/AVIF 等高效图片格式，在保证画质的前提下减少 50% 以上的文件体积
- 响应式图片，根据用户设备屏幕加载不同尺寸的图片，避免小屏加载大图浪费带宽
- 图片懒加载，非首屏图片滚动到可视区域再加载，优先加载首屏核心内容

**代码优化**
- 代码压缩与 Tree Shaking，移除未使用的代码，减少 bundle 体积
- 路由级代码分割：C 端统一应用内，按功能模块做路由级懒加载，将大 bundle 拆分为小文件，按需加载，首屏仅加载主应用核心代码，其他模块访问时再加载，保证首屏速度
- 智能分包策略：将第三方库和业务代码分离，利用浏览器缓存机制，提升重复访问的速度

**资源加载优化**
- CDN 分发静态资源，将图片、JS、CSS 等静态资源部署到 CDN，降低网络延迟
- 合理设置缓存策略，静态资源设置长缓存，动态资源设置短缓存，减少重复请求
- 预加载关键资源，提前加载用户可能需要的资源，提升后续页面的加载体验

**首屏优化**
- 首屏加载时间控制在 3 秒以内，这是用户留存的临界值
- 采用服务端渲染 (SSR) 或静态生成 (SSG)，提升首屏渲染速度，改善 SEO
- 内联关键 CSS，避免 CSS 阻塞渲染，减少白屏时间

### 4.4 响应式适配规范
- **标准断点设计**：设置统一的断点：360px (移动端)、768px (平板)、1200px (桌面)、1440px (大屏)，覆盖绝大多数用户设备。
- **弹性布局**：使用 Flexbox 和 Grid 布局，实现模块的自适应调整，避免固定像素布局。
- **适配细节**：针对不同断点调整字体大小、间距、布局，确保小屏下内容不拥挤，大屏下内容不松散，所有设备下体验一致。

## 五、电商特有功能规范

### 5.1 商品展示规范
- **高清原创素材**：使用高清原创图片和视频，避免模糊、压缩过度的素材，高端品牌尤其注重图片的质感和调性，匹配品牌的定位。
- **场景化展示**：不仅展示商品本身，还要展示使用场景，让用户感知产品带来的生活方式，如梵几的家居场景图，让用户想象产品放在自己家里的样子。
- **工艺细节传递**：展示材质、工艺细节，突出产品的品质，如木墨的工厂系列视频，展示手工制作的过程，传递品牌的匠心。

### 5.2 购物车与结算规范
- **迷你购物车**：导航栏 hover 时显示迷你购物车，用户可以快速查看、修改商品，无需进入购物车页面，提升操作效率。
- **简化结算流程**：将结算流程步骤控制在 3 步以内，支持游客购买，无需强制用户注册，减少用户的流失。
- **透明价格体系**：价格展示清晰透明，包含运费、税费等所有费用，没有隐藏费用，提升用户的信任度。

### 5.3 搜索与筛选规范
- **智能搜索**：支持模糊搜索、拼音搜索，用户输入时实时提示搜索结果，提升搜索的效率和准确率。
- **精简筛选维度**：高端品牌避免过多的筛选条件，只保留核心的筛选维度，避免界面过于复杂，干扰用户。
- **友好的空结果反馈**：当搜索无结果时，给出友好的提示，同时推荐热门商品，引导用户继续浏览。

### 5.4 品牌内容营销规范
- **内容化电商**：融入品牌故事、栏目、视频等内容，不仅仅是卖货，更是传递品牌理念，如梵几的 "梵几在家" 栏目，木墨的 "双手做工" 市集，通过内容建立用户的品牌认同。
- **社交生态融合**：展示小红书、微信、Instagram 等社交平台入口，引导用户关注品牌账号，沉淀私域流量，提升品牌的长期影响力。
- **线上线下融合**：展示线下门店信息，支持线上预约、线下体验，实现线上线下的融合，满足高端用户的体验需求。

## 六、可访问性与兼容性

### 6.1 可访问性规范
- **颜色对比度**：文字与背景的对比度符合 WCAG 2.1 标准，确保视力障碍用户可以正常阅读内容。
- **全键盘导航**：支持全键盘操作，用户可以通过 Tab 键导航到所有可交互元素，满足不同用户的需求。
- **语义化支持**：通过语义化 HTML 标签，让屏幕阅读器可以正确识别页面的结构和内容，提升视障用户的体验。
- **图片 Alt 文本**：所有图片都添加描述性的 alt 文本，不仅提升可访问性，也有利于 SEO 优化。

### 6.2 兼容性规范
- **浏览器兼容**：兼容所有主流浏览器：Chrome、Firefox、Safari、Edge，确保不同浏览器下的体验一致，避免兼容性问题。
- **设备兼容**：适配不同尺寸的手机、平板、桌面设备，覆盖 99% 以上的用户设备，确保所有用户都能正常访问。
- **弱网兼容**：针对弱网环境进行优化，提供离线缓存机制，确保用户在弱网环境下也能正常浏览已加载的内容。

## 七、不同品牌定位的设计差异参考

根据你的品牌定位，可以调整设计的侧重点，参考这些品牌的差异：
- **高端家居品牌**（梵几、木墨、ZENS）：侧重东方美学、自然质感，突出生活方式和品牌故事，设计更温润、克制，注重内容的叙事性。
- **高端时尚品牌**（之禾、Prada）：侧重极简轻奢、视觉冲击，突出品牌调性和产品质感，设计更简洁、高级，注重视觉的沉浸感。
- **潮流买手店**（Machine-A）：侧重潮流活力、商品效率，突出品牌集合和促销信息，设计更灵活、直接，注重交易的转化率。

## 八、屿音多应用统一设计规范

针对屿音电商最新的「C 端统一前台 + B 端独立中台」前端架构，所有应用需遵循统一的设计系统，确保全平台体验一致、组件复用，降低开发成本，同时针对不同应用的用户场景做针对性优化：

### 8.1 C 端统一前台应用规范（Web / H5 / 小程序）

Web 端作为所有 C 端用户的唯一访问入口（桌面端 + 移动端 H5 同一 React 应用，360px 断点自适应），与微信小程序共享同一套业务逻辑、API、组件。**所有 C 端功能都在同一个 yuyin-web 仓内**：
- 统一顶部导航，整合所有 C 端功能的快捷入口，用户通过路由跳转无缝切换功能，无任何加载延迟
- 统一全局状态管理：用户信息、购物车、会话、营销活动等全局状态统一管理（Zustand），全模块共享，实现跨功能的实时数据同步
- 统一 API 层：封装对接所有后端微服务的统一 API（`src/api/`），避免重复请求、重复封装，提升接口调用效率
- 品牌调性优先，突出屿音的品牌定位，强化内容化电商的叙事能力

**内部功能模块**（均在本仓 `src/features/` 下）：
- **用户中心**：极简双栏布局，信息分层展示，移动端导航自动折叠为底部标签栏（`MobileLayout`），保证单手操作
- **购物系统**：迷你购物车、极简操作流程，操作结果即时反馈
- **支付金融中心**：安全优先，界面极致简洁，支付信息透明展示，状态清晰反馈
- **订单中心**：统一订单卡片，状态直观区分，操作按钮精简，详情页信息流符合用户浏览习惯
- **营销活动中心**：活动卡片统一尺寸，克制营销，简化参与流程，规则轻量化展示
- **消息与客服中心**：消息列表极简，对话界面简洁，自助服务折叠展示

### 8.2 跨端共享资产
- **Figma 设计令牌**（颜色 / 字体 / 间距 / 圆角）：通过 Figma API 拉取，自动生成 CSS Variables
- **后端 API 契约**（OpenAPI/Proto）：与小程序、App 共享同一套接口
- **品牌资产**（字体 / 启动图 / 视频）：自托管在 `public/`

## 九、移动端 + 微信小程序双端适配规范

Web 端内置桌面端 + 移动端 H5 双布局（`DesktopLayout` / `MobileLayout`），移动端 H5 直接复用 Web 端代码，360px 唯一断点自适应。微信小程序为独立仓 `wushiboW/yuyin-miniprogram`，三端共享同一套设计规范。

### 9.1 移动端 H5 专属设计开发规范

**布局适配规则**
- 固定移动端断点：唯一移动端适配断点 360px，所有手机设备统一适配，杜绝多尺寸错乱布局
- 全屏流式布局：取消桌面端多列宽布局，采用单列流式结构，模块全屏适配，左右留白统一固定间距
- 底部操作优先：核心操作（加购、下单、支付、提交）固定页面底部，适配移动端单手触控习惯
- 导航极简重构：顶部仅保留 LOGO、搜索、个人中心，一级导航折叠为侧边弹出菜单，底部增加核心功能 Tab 栏（首页、分类、购物车、我的）

**交互适配规则**
- 触控尺寸规范：所有可点击按钮、图标最小触控尺寸 ≥ 44px，符合移动端人机交互标准
- 手势交互优化：支持左右滑动切换商品图、下拉刷新、上拉加载更多、右滑返回上一页
- 弹窗适配：所有弹窗移动端全屏/居中适配
- 输入体验优化：表单输入框自动唤起对应键盘，输入框自动上移规避键盘遮挡

**性能专项优化**
- 移动端专属图片压缩，优先加载 WebP/AVIF 格式
- 开启移动端专属懒加载、预加载策略
- 禁用移动端冗余动效，保留轻量化渐显、过渡动画

### 9.2 微信小程序（独立仓 yuyin-miniprogram）

- 状态栏自适应、底部 Tab 原生配置
- 主包 < 2MB，分包加载
- 微信原生支付、登录、授权、分享、推送能力优先
- 详细规范见 `wushiboW/yuyin-miniprogram/CLAUDE.md`（待创建）
